    steg1 login 部分

    反汇编能够看到有两个 steg，其中 steg2 要在 steg1 成功 login 才能进入。关于 steg1 的漏洞，能发现在 show status 即 sub_195B() 的功能有一个格式化字符串漏洞。

    printf(format_, a2, v2, v3, v4, v5, v7, v8, STACK_ANCHOR_[0], STACK_ANCHOR_[1], v10, v11);。通过这个格式化字符串漏洞，我们能成功泄露出 login 时的 pwd。并且这个漏洞只能使用一次，因为第一次结束后程序会把 dword_52C4 = 1;，导致下一次进行 show 时，程序会进入 if 分支。下面给出 pwndbg 的调试流程，因为比较长，所以不截图了。

    现在先来分析一下这个流程，因为这是复现，所以写的比较详细（现场打不出来太菜了QAQ）。

    因为要调试格式化字符串的偏移，所以我们断点断在 $piebase + 195b = 0x55555555595b$ 处，因为 show 的偏移在 0x195b 处（这里不知道为什么调试显示的这一步消失了）。进入 login（sub_22D9()），可以看到 snprintf(s, 0x28u, "%016lx%016lx", qword_52D0, qword_52D8);，最后比较的 pwd 是存在 qword_52D0 和 qword_52D8 偏移处的。而通过上述调试时的反汇编可以看到：

    qword_52d0 被放到了 [rbp-0x40]，qword_52d8 被放到了 [rbp-0x38]。而在函数开头有 sub rsp, 0x40，所以在 call printf@plt 之前：rsp == rbp - 0x40。所以当前 [rsp] = [rbp-0x40] = qword_52d0，[rsp+0x8] = [rbp-0x38] = qword_52d8。既然存储在 rsp 当中，那么偏移取到 6 和 7 就可以得到存储的 pwd 了。虽然取到 pwd 了，但为了后续的攻击，在这里先取出 PIE 和 libc 相关的值来得到对应的基址，即第 15 个位置和第 23 个位置。

    按照上面的调试我们可以得到 pwd：0x8056d91b35ec6265 0x2a0fc9803c69868f、PIE 相关地址：0x0000555555556893、libc 相关地址：0x00007ffff7c29d90、0x7ffff7c00000 0x7ffff7c28000 r--p 28000 0 /usr/lib/x86_64-linux-gnu/libc.so.6，减去基址得到偏移即可。

    $pieoff = 0x0000555555556893 - 0x555555554000 = 0x2893$

    $libcoff = 0x00007ffff7c29d90 - 0x7ffff7c00000 = 0x29d90$

    再总结一遍：

    1. 用格式化字符串泄露登录密码、PIE、libc：%6$p、%7$p、%15$p、%23$p。
    2. 登录后利用 off-by-one 做堆重叠。
    3. 伪造 fake chunk，打 unlink，拿到改 .bss qword_5140 表的能力。
    4. 伪造 House of Apple2 需要的三块堆结构。
    5. 先改 _IO_list_all，再把 qword_5140[2] 改成 pivot 区。
    6. 写 fake stack + shellcode，退出触发。
